# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a build tool that works
with multi-package repos, or single-package repos to help you version and publish packages.

## What is a changeset?

A changeset is a piece of information about changes made in a branch or commit. It is a markdown file with
frontmatter that describes the changes and how they should be released.

When you make changes that should be released, run `pnpm changeset` to create a changeset file. When the
changes are merged to master, changesets will open a pull request to version and publish the packages.

## When to add a changeset?

You should add a changeset when you make changes that should trigger a release. This typically means any
user-facing change, but can also include internal changes that affect the published API.

## How to add a changeset?

Run `pnpm changeset` and follow the prompts.
